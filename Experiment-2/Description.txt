In this experiment-3, we have created a DMIN DASHBOARD with light and dark mode theme.

ADMIN DASHBOARD DESCRIPTION & STRUCTURE

1.HEADER
Purpose:Displays the title of the dashboard and provides a toggle for Dark Theme mode.
HTML Elements Used:

<header — Wraps the title and theme toggle.
<h1 — Displays the text "Admin Dashboard".
<label and<input type="checkbox" — Creates a labeled toggle for Dark Theme.
Styling Highlights:

display: flex; justify-content: space-between; align-items: center; — Ensures the title and theme toggle are horizontally aligned and spaced apart.
Header background color changes dynamically based on theme mode (light/dark).

2. SIDEBAR (Navigation Menu)
Purpose: Provides quick navigation links to different sections (Home, Users, Settings).
HTML Elements Used:

<nav class="sidebar — Contains the vertical menu.
<ul> + <li> + <a> — Creates a list of navigation links.
Styling Highlights:

Fixed width column in the grid layout (grid-area: sidebar).
Light or dark background color changes with theme toggle.
Borders (border: 1px solid black) separate the sidebar visually.
Links styled with color: var(--link-color); text-decoration: none; for a clean look.

3. MAIN CONTENT

Purpose: Displays the welcome message and placeholder for charts, tables, or reports.
HTML Elements Used:

<main> — The main content container.
<h3> — Welcome message.
<p> Instruction text.
Styling Highlights:

grid-area: main— Occupies the main section in the grid layout.
Padding (padding: 2rem;) ensures text and elements are not cramped.
Border around content for separation from sidebar.

4. FOOTER

Purpose: Displays copyright.
HTML Elements Used:

  <footer> — Footer container.
  <p> — Footer text.
Styling Highlights:

grid-area: footer — Occupies the full bottom row in grid layout.
White text on a dark background for strong contrast.
Centered text (text-align: center;).

CSS STYLING BREAKDOWN

Theme Variables

Uses \:root to define color variables for background, text, header, footer, sidebar, main content, and link colors.
Dark Theme enabled when #darktheme is checked using:

  
  body:has(#darktheme:checked) {
    --bg-color: #1a1a1a;
    --text-color: #f0f0f0;
    
  }
  
   This approach avoids rewriting multiple style rules; only variables change.


2. Page Layout

Grid Layout:

Two columns: 160px for sidebar, rest for content.
Three rows: header (auto), main (1fr), footer (auto).
Layout areas: "header header" "sidebar main" "footer footer".


3. Typography & Defaults

Font: sans-serif.
Color: Controlled by --text-color(changes with theme).
Margins & paddings reset with:


  * {
    margin: 2;
    padding: 0;
    box-sizing: border-box;
  }
  

4. Dark Theme Toggle Styling

Checkbox scaled up (transform: scale(1.2)) for better visibility.
Accent color matches link color (accent-color: var(--link-color);).


LEARNING OUTCOMES
1.How to create a complete Admin Dashboard layout using HTML and CSS Grid.
2.How to store and apply CSS variables for theme customization.
3.How to implement a dark theme toggle using only CSS and a hidden checkbox.
4.How to structure HTML semantically for better organization.
